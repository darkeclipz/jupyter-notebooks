    \hypertarget{definitions}{%
\section{Definitions}\label{definitions}}

A genetic algorithm is essentially an improved brute-forced method to
find the optimal, or most \emph{fit}, set of \emph{variables} within a
set of allowed variables and their \emph{domains}. We will first lay out
the terminology that comes with genetic algorithms.

    \hypertarget{genotype}{%
\subsection{Genotype}\label{genotype}}

A genotype, is a variable, which in term is a changable entry in the
DNA, which has a limited set of allowed values, which are called it's
\emph{domain}.

For example, say we have a domain of all the letters, which we call
\(G = \{A,B,C,D,\ldots,Z\}\). The genotype will hold a value
\(g \in \mathbb{Z}\) such that the value corresponds to an entry in
\(G\). More concrete, if \(g=2\) then the corresponding element in \(G\)
is \(B\).

    \hypertarget{dna}{%
\subsection{DNA}\label{dna}}

The DNA of the sample holds all the values of the genotypes. This is
defined as the set \(D = \{ g_0, g_1, g_2, \ldots g_{n-1}, g_n\}\) where
\(g\) is a genotype.

    \hypertarget{population}{%
\subsection{Population}\label{population}}

The algorithm starts with a population of \(N\) elements, where the
\emph{DNA} of the sample is defined with uniform probability at the
start.

If for example \(N=1000\), we create a set
\(P=\{d_1,d_2,d_3,\ldots,d_{1000}\}\), where \(d\) is defined as DNA.
All the genotypes in each DNA will point to a random value in domain of
the genotype.

    \hypertarget{fitness}{%
\subsection{Fitness}\label{fitness}}

A certain configuration of DNA should be measured for \emph{fitness},
which means how good this DNA scores. This is used to fill the mating
pool, and ensures survival of the fittest.

\hypertarget{fitness-function}{%
\subsubsection{Fitness function}\label{fitness-function}}

For our example, where each genotype in the DNA represents a letter, we
want to match a certain sentence. Therefore we can define our fitness
function as:

\[ f(\text{correct words}) = \text{correct words}\]

which will be an integer value between \([0, \text{sentence length}]\).

\hypertarget{fitness-curve}{%
\subsubsection{Fitness curve}\label{fitness-curve}}

Instead of using a linear function, we can also apply a curve to the
fitness function. Here is a list of basic functions which can be used to
create a curve:

\begin{itemize}
\tightlist
\item
  Linear (yes, not a curve): \(x\).
\item
  Power: \(a^x\).
\item
  Exponential: \(e^x\).
\item
  Square-root: \(\sqrt{x}\).
\item
  Logarithmic: \(\ln x\).
\end{itemize}

    \hypertarget{mating-pool}{%
\subsection{Mating pool}\label{mating-pool}}

Based on the fitness, calculated with \(f\), of each DNA, we want to
create a mating pool. This can be done in two ways: a
\emph{probabilistic} method or a \emph{rank} based method.

\hypertarget{probabilistic-method}{%
\subsubsection{Probabilistic method}\label{probabilistic-method}}

With this method, we normalize the fitness for each DNA, expressed as
\(f_{norm}\):

\[ f_{norm} = \dfrac{ f(d) }{\sum\limits_{g\in D} f(d)} \]

As an example when there are only \(4\) samples in the population. First
we calculate \(\sum_{g\in D} f(d)\), which in this case, is \(26\).

\begin{longtable}[]{@{}lll@{}}
\toprule
Sample & Fitness score & Fitness score (normalized)\tabularnewline
\midrule
\endhead
1 & 12 & \$\dfrac{12}{26} \approx  \$ 0.462\tabularnewline
2 & 7 & \$\dfrac{7}{26} \approx  \$ 0.269\tabularnewline
3 & 5 & \$\dfrac{5}{26} \approx  \$ 0.192\tabularnewline
4 & 2 & \$\dfrac{2}{26} \approx  \$ 0.077\tabularnewline
\bottomrule
\end{longtable}

\hypertarget{rank-method}{%
\subsubsection{Rank method}\label{rank-method}}

In some cases, where \(f_{norm}\) ends up to be \(0.98\) (\(98\%\)),
sorting them by rank can fix the issue of overselecting. Simply sort all
the elements descending by rank (ascending on fitness score), and divide
the rank by \(\dfrac{n(n+1)}{2}\) where \(n\) is the number of ranks.

As an example, we have a population where one DNA scores very high.
First we find the sum of rank numbers, there are \(4\) samples, so we
find:

\[ \dfrac{n(n+1)}{2} = \dfrac{4\cdot 5}{2} = 10 = 1+2+3+4\]

Then we sort them by rank, and normalize the rank score:

\begin{longtable}[]{@{}lllll@{}}
\toprule
Sample & Fitness score & Fitness score (normalized) & Rank & Rank
(normalized)\tabularnewline
\midrule
\endhead
1 & 20 & \(\dfrac{20}{24} \approx 0.833\) & 4 &
\(\dfrac{4}{10} = 0.4\)\tabularnewline
2 & 3 & \(\dfrac{3}{24} = 0.125\) & 3 &
\(\dfrac{3}{10}=0.3\)\tabularnewline
